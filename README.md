# ArrivePack — V1 MVP

> **Plan first. Travel smarter.**

ArrivePack is a premium mobile travel companion app. V1 focuses on the **Egypt Pack** — helping users prepare before an international trip and supporting them during the trip with live, location-aware alternatives.

---

## How to Run

### Prerequisites
- Node.js 18+
- Expo Go app on iOS or Android (for device preview)
- OR run on iOS Simulator / Android Emulator

### Install & Start

```bash
cd arrive-pack
npm install
npx expo start
```

Then:
- **iOS Simulator**: press `i`
- **Android Emulator**: press `a`
- **Physical device**: scan the QR code with Expo Go

## Unsupported Destination Backend Setup

When a user searches a destination that has no verified pack (e.g. Italy, Japan, Tokyo), ArrivePack offers a basic AI-assisted planning draft with Supabase database caching.

### How the caching works

1. User taps **Create basic planning draft** for Italy.
2. App calls the Supabase Edge Function `create-basic-draft`.
3. Edge Function checks `destination_basic_drafts` table for slug `italy`.
4. **Cache miss** → calls OpenRouter, saves result to DB, returns draft with `provider: "openrouter"`.
5. **Cache hit** → returns DB draft immediately, does NOT call OpenRouter. Returns `provider: "database_cache"`.

The UI shows a small badge: **Generated by AI**, **Saved draft**, or **Mock draft for MVP**.

This is verifiable from OpenRouter usage logs — only unique destination slugs will show up.

### 1. Run database migrations

```bash
supabase db push
# or apply manually in the Supabase SQL editor
```

Migrations are in `supabase/migrations/`.

### 2. Set Expo environment variables

Copy `.env.example` to `.env.local` and fill in:

```txt
EXPO_PUBLIC_SUPABASE_URL=https://<your-project>.supabase.co
EXPO_PUBLIC_SUPABASE_ANON_KEY=<your-anon-key>
EXPO_PUBLIC_FORMSPREE_ENDPOINT=https://formspree.io/f/mrevkwel
```

> `EXPO_PUBLIC_` variables are safe to include in the mobile bundle. Never put the OpenRouter key here.

### 3. Set Supabase Edge Function secrets

After logging in with the Supabase CLI:

```bash
supabase secrets set OPENROUTER_API_KEY=<your-openrouter-key>
supabase secrets set OPENROUTER_MODEL=openai/gpt-3.5-turbo
supabase secrets set FORMSPREE_ENDPOINT=https://formspree.io/f/mrevkwel
```

`SUPABASE_URL` and `SUPABASE_SERVICE_ROLE_KEY` are injected automatically — no need to set them.

Use a cost-effective model (e.g. `openai/gpt-3.5-turbo`, `mistralai/mistral-7b-instruct`, `google/gemma-2-9b-it:free`).

### 4. Deploy the Edge Function

```bash
supabase functions deploy create-basic-draft
```

### 5. Test end-to-end

1. Search for "Japan" → tap **Create a basic planning draft**
   - Loading state → draft panel appears
   - Badge shows **Generated by AI**
   - `destination_basic_drafts` table has a new row
   - Formspree email has `action: basic_draft_created`, `provider: openrouter`
2. Search "Japan" again → tap **Create a basic planning draft**
   - Returns instantly (no OpenRouter call)
   - Badge shows **Saved draft**
   - Formspree email has `action: basic_draft_used`, `provider: database_cache`
3. Tap **Request Japan Pack**
   - Formspree email has `action: request_pack`

### Formspree email actions summary

| Action | When |
|--------|------|
| `basic_draft_created` | New draft generated by OpenRouter and saved |
| `basic_draft_used` | Existing cached draft served from DB |
| `request_pack` | User requested a verified pack |

### Graceful fallback (no backend configured)

If `EXPO_PUBLIC_SUPABASE_URL` or `EXPO_PUBLIC_SUPABASE_ANON_KEY` are not set, the app uses a local mock draft automatically (badge shows **Mock draft for MVP**). Pack requests still send directly to Formspree. The app never crashes due to missing configuration.

## Architecture Notes

- All data is **local mock data** — no real API calls for the Egypt Pack
- Services are abstracted (`readinessEngine`, `itineraryEngine`, `rescueEngine`) so real APIs can be swapped in
- Unsupported destination service: `src/services/unsupportedDestinationService.ts`
- Supabase Edge Function: `supabase/functions/create-basic-draft/index.ts`
- Zustand store persists to AsyncStorage automatically
- TypeScript strict mode — zero type errors
- Future API integration points: Visa APIs, Places APIs, eSIM providers, booking affiliates, weather, currency exchange
